# 剧本创作流程 UI/UX 重构规范说明文档

本文档包含了剧本创作流程“工作台 + 对话式”布局重构的详细规范，包括节点流程图、样式、动画时序以及 QA 验收交互走查表。

---

## 1. 整体布局规范 (Workbench + Conversational)

### 1.1 左侧工作台 (Left Panel)
- **宽度**: 默认 280px，支持通过拖拽句柄调整 (范围 200px - 600px)。
- **折叠状态**: 宽度缩至 48px，隐藏文字，仅保留图标（Tooltips）。
- **包含模块**: 大纲 (Outline)、素材库 (Assets)、历史记录 (History)。
- **样式**: `8px` 圆角，`0 4px 12px rgba(0,0,0,0.08)` 阴影，白色背景。

### 1.2 中央画布 (Center Canvas)
- **对话气泡模式**: 模拟聊天流，从底部向上滑入。用户内容右侧蓝底，AI 内容左侧白底。
- **剧本文字模式**: 采用 Tiptap 富文本编辑器，支持手动编辑、复制粘贴，展示字数和预计阅读时长统计，并提供撤销重做（Ctrl+Z / Ctrl+Y）。
- **样式**: 宽度自适应，背景 `#F8FAFC`。

### 1.3 右侧属性/AI面板 (Right Panel)
- **宽度**: 固定 320px。
- **显示切换**: 支持 `Ctrl+\` 快捷键切换显隐。
- **包含模块**: 
  - **快速设定**: 默认展开，最高 320px，支持折叠。
  - **AI 助手**: 下半部分占满剩余高度。

## 2. 动画时序图与缓动函数 (Animations)

| 动画对象 | 触发时机 | 持续时间 | 缓动函数 (Cubic Bezier) | 关键帧参数 |
|---------|----------|---------|-------------------------|-----------|
| 侧边栏折叠 | 点击折叠按钮 | 0.3s | `ease` (0.25, 0.1, 0.25, 1) | `width: 280px -> 48px` |
| 节点出现 | 章节新增 | 0.35s | `cubic-bezier(0.34, 1.56, 0.64, 1)` | `scale: 0.8->1`, `opacity: 0->1`, `translateY: 20px->0` |
| 节点消失 | 章节删除 | 0.35s | `cubic-bezier(0.34, 1.56, 0.64, 1)` | `scale: 1->0.8`, `opacity: 1->0`, `translateY: 0->-20px` |
| 节点回弹 | 拖拽松手 | 0.2s | `cubic-bezier(0.34, 1.56, 0.64, 1)` | `translateY: 10px -> 0` |
| SVG 连线 | 初始化/开启开关 | 0.6s | `ease-out` (0, 0, 0.2, 1) | `transform: scaleY(0) -> scaleY(1)` |
| 气泡滑入 | 发送/接收消息 | 0.3s | `ease-out` (0, 0, 0.2, 1) | `translateY: 40px -> 0`, `opacity: 0->1` |

## 3. 剧本文字样式 (Script Typography)

当在中央画布切换为“剧本文字”模式，或节点中激活此样式时，排版遵循以下标准：
- **字体族**: `"Courier New", Courier, "Songti SC", serif`
- **字号**: `16px`
- **行距**: `1.8`
- **字重**: `500` (Medium)
- **颜色**: `#1e293b` (Slate-800)
- **左边距**: `24px` (针对节点内容缩进)
- **高亮**: 章节/场景标题采用 `#4f46e5` (Indigo-600) 且加粗 (`font-bold`)。

## 4. 节点流程图类型说明 (Node Types)

虽然当前实现为“章节大纲”垂直列表，但若后续扩展为全流程图，四种节点规范如下：

1. **角色节点**:
   - 颜色: `#3b82f6` (Blue)
   - 图标: 👤 User
   - 规则: 仅允许配置人物档案，输出传递给条件节点。
2. **条件节点**:
   - 颜色: `#eab308` (Yellow)
   - 图标: 🔀 Split
   - 规则: 必须包含 ≥2 个输出分支，判定用户选择或随机率。
3. **跳转节点**:
   - 颜色: `#8b5cf6` (Indigo)
   - 图标: ↪️ Redirect
   - 规则: 不包含具体剧情内容，仅指向目标章节 ID。
4. **结束节点**:
   - 颜色: `#ef4444` (Red)
   - 图标: ⏹️ Stop
   - 规则: 仅有输入，无输出分支，展示结局字幕。

## 5. 交互走查表 (QA Checklist)

请 QA 团队按照以下 20 条核心用例进行验收：

| 编号 | 测试模块 | 测试步骤 | 预期结果 | 通过标准 |

| 1 | 左侧面板 | 拖动右侧边缘句柄 | 宽度在 200px - 600px 间平滑调整 | 无卡顿，宽度受限 |
| 2 | 左侧面板 | 点击“<”折叠按钮 | 面板缩小至 48px，隐藏文字 | 动画 0.3s，保留图标 |
| 3 | 快捷键 | 键盘按下 `Ctrl+\` | 右侧面板隐藏/显示切换 | 动画过渡平滑 |
| 4 | 大纲 Tabs | 切换“素材库”与“大纲” | 内容区域正确渲染对应状态 | 激活 Tab 有蓝色下划线 |
| 5 | AI 输入模式 | 点击“故事梗概”旁边的魔法棒图标 | 弹出 300px 宽的输入框 (Popover) | Popover 定位准确 |
| 6 | AI 生成 | 在输入框输入文字，点击“追加” | 原文末尾增加生成的 Mock 文本 | 触发 ElMessage 提示成功 |
