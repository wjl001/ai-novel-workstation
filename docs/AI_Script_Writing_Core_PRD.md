## 0. 设计综述 (Executive Summary)

本PRD聚焦于 **AI Script Writing** 核心模块，旨在打造一个符合专业编剧工业标准的智能化创作工作流。
系统将剧本创作拆解为四个严谨的工序：**项目管理** -> **创意孵化** -> **结构搭建** -> **沉浸撰写**。
通过AI技术（LLM + RAG），在保持人类创作者核心创意主导权的前提下，指数级提升剧本生产效率，实现从“灵感”到“拍摄蓝图”的无缝转化。

## 1. 模块一：项目管理中心 (Project Hub)

**页面定位**：创作者的“个人工作室”。管理所有剧本项目的入口，提供快速的新建、封面生成和状态预览功能。
**对应页面**：`src/views/AIScriptWriting/Projects.vue`

### 1.1 页面布局与交互 (Page Layout)
*   **顶部导航栏**：
    *   **标题**：显示 "我的剧本工作室"。
    *   **搜索框**：支持按标题关键词过滤项目。
    *   **排序按钮**：支持按“最近编辑”、“创建时间”排序。
*   **新建项目按钮 (Primary Button)**：
    *   *位置*：页面右上角。
    *   *交互*：点击后立即创建一个空的 `Project` 对象，并直接跳转至 `AIWriteNovel.vue`（创意孵化页），URL携带新生成的 `id`。

### 1.2 项目卡片 (Project Card)
每个项目以卡片形式展示，分为三个区域：
1.  **封面区 (Cover Area)**：
    *   *显示*：项目封面图。若无图，显示默认占位符。
    *   *悬浮交互*：鼠标悬停时，显示半透明遮罩层，包含两个圆形按钮：
        *   **编辑按钮 (Icon: Edit)**：点击跳转至编辑器。
        *   **魔法棒按钮 (Icon: MagicStick)**：点击打开“AI封面生成器”弹窗。
2.  **信息区 (Info Area)**：
    *   *标题*：显示剧本标题（单行截断）。
    *   *元数据*：显示“总字数”和“最后更新时间”。
3.  **操作区 (Action Area)**：
    *   *更多菜单 (Three Dots)*：点击弹出下拉菜单，包含“删除项目”、“导出备份”、“重命名”。

### 1.3 AI 封面生成器弹窗 (Cover Generator Modal)
*   **触发**：点击卡片上的魔法棒按钮。
*   **界面元素**：
    *   **提示词输入框**：预填默认提示词 `"${title}" movie poster, cinematic lighting, high detail`。用户可修改。
    *   **生成按钮**：点击后进入 Loading 状态，调用绘图 API。
    *   **结果预览区**：
        *   显示生成的 4 张预览图。
        *   点击某张图，高亮选中。
    *   **确认按钮**：点击后将选中的图片 URL 保存为项目封面，并关闭弹窗。

## 2. 模块二：创意孵化台 (Concept Incubator)

**页面定位**：剧本的“灵魂”诞生之地。定义世界观、角色、题材和核心冲突。
**对应页面**：`src/views/AIScriptWriting/AIWriteNovel.vue`

### 2.1 页面结构 (Page Structure)
*   **顶部步骤条 (Step Indicator)**：高亮显示当前为“第一步：基础设定”。
*   **主内容区**：分为左右两栏（或上下分块）。

### 2.2 基础设定表单 (Core Settings Form)
用户需填写以下字段，确立剧本的工业规格：

1.  **剧本标题 (Title)**：
    *   *交互*：文本输入框。支持“AI 随机取名”按钮，点击生成 5 个备选标题供选择。
2.  **核心题材 (Genre)**：
    *   *交互*：级联选择器 (Cascader)。
    *   *选项*：玄幻(东方/高武)、都市(职场/豪门)、悬疑(刑侦/无限流)等。
3.  **目标受众 (Target Audience)**：
    *   *交互*：单选按钮组 (Radio Group)。
    *   *选项*：男频爽文、女频言情、全年龄向。
4.  **剧本规格 (Specs)**：
    *   **预计集数**：数字步进器 (Stepper)，默认 80。
    *   **单集时长**：数字输入框，单位“分钟”，默认 1.5。
5.  **风格标签 (Tags)**：
    *   *交互*：点击“添加标签”按钮，弹出标签选择器。
    *   *预设标签*：快节奏、反转、甜宠、虐心、黑色幽默。
    *   *自定义*：支持输入新标签按回车添加。

### 2.3 创意输入区 (Creative Input)
1.  **一句话梗概 (Logline)**：
    *   *交互*：多行文本框。
    *   *AI 润色按钮*：点击后，AI 分析输入内容，扩充为更具吸引力的“钩子”文案（如增加冲突、明确主角目标）。
2.  **核心金手指 (Golden Finger)**：
    *   *交互*：文本框。输入主角的特殊能力（如：透视眼、系统、重生）。
3.  **世界观设定 (World View)**：
    *   *交互*：富文本编辑器。支持输入物理规则、魔法体系等。

### 2.4 底部操作栏 (Footer Actions)
*   **下一步按钮 (Next Step)**：
    *   *交互*：点击后进行表单校验。
    *   *逻辑*：校验通过后，保存数据至 `loreStore`，路由跳转至 `NovelGenerator.vue`。

## 3. 模块三：结构工程台 (Structure Architect)

**页面定位**：剧本的“骨架”搭建。从大纲生成到分集管理。
**对应页面**：`src/views/AIScriptWriting/NovelGenerator.vue`

### 3.1 状态一：大纲生成 (Outline Generation)
当 `currentStep === 'outline'` 时显示此视图。

#### 3.1.1 自动生成面板
*   **加载动画**：进入页面即自动触发生成，显示“AI 正在构建故事骨架...”的 Lottie 动画。
*   **流式展示区**：
    *   模拟打字机效果，逐行输出章节标题。
    *   *交互*：用户无法编辑，只能观看，直到生成结束。

#### 3.1.2 大纲预览与编辑
生成完成后，显示章节列表：
*   **列表项 (List Item)**：
    *   **序号**：CH.1
    *   **标题输入框**：用户可修改生成的标题。
    *   **梗概输入框**：用户可修改生成的剧情简介。
    *   **操作按钮**：
        *   **重生成 (Refresh)**：仅重写该章节的大纲。
        *   **删除 (Trash)**：移除该章节。
*   **底部工具栏**：
    *   **添加章节**：在末尾追加一个空章节。
    *   **重新生成全部**：清空当前所有结果，重新调用 AI。
    *   **确认大纲 (Confirm)**：
        *   *交互*：点击后弹出确认框“确认后将进入分集写作模式”。
        *   *逻辑*：将大纲数据固化，`currentStep` 变更为 `chapters`。

### 3.2 状态二：章节管理 (Chapter Management)
当 `currentStep === 'chapters'` 时显示此视图。

#### 3.2.1 统计仪表盘 (Dashboard)
顶部展示三个数据卡片：
1.  **总章节**：显示当前章节总数。
2.  **已完成**：显示状态为 `completed` 的章节数。
3.  **批量操作入口**：
    *   **“一键对接 - 批量短剧”卡片**：
        *   *交互*：点击打开 `ScriptDialog`（批量模式）。
        *   *功能*：将所有章节的大纲/正文一次性导出为分镜脚本。

#### 3.2.2 章节列表 (Chapter List)
*   **列表项详情**：
    *   **状态标签**：未开始（灰色）/ 进行中（蓝色）/ 已完成（绿色）。
    *   **AI 撰写按钮**：
        *   *交互*：点击跳转至 `Editor` 页面，并自动触发 AI 开始写正文。
    *   **手写按钮**：
        *   *交互*：点击跳转至 `Editor` 页面，光标定位到正文起始处。
    *   **对接短剧按钮 (Video Icon)**：
        *   *交互*：点击打开 `ScriptDialog`（单集模式）。

### 3.3 短剧剧本导出弹窗 (Script Export Modal)
*   **预览区**：
    *   左侧显示生成的标准分镜脚本（包含场景、人物、动作、对白）。
    *   支持简单的文本编辑。
*   **项目选择器**：
    *   下拉框列出下游视频制作平台的项目列表。
*   **操作按钮**：
    *   **推送草稿**：调用 API 将脚本发送至选中项目。
    *   **复制到剪贴板**：一键复制文本。
    *   **下载文件**：导出为 `.txt` 或 `.docx`。

## 4. 模块四：沉浸撰写台 (Immersive Editor)

**页面定位**：文字生产车间。集成 AI 辅助、角色管理和实时优化的富文本编辑器。
**对应页面**：`src/views/AIScriptWriting/Editor.vue`

### 4.1 界面布局 (Layout)
采用经典的“左-中-右”三栏布局：
1.  **左侧栏 (Sidebar - Lore)**：展示大纲导航和角色库。
2.  **中间栏 (Main - Editor)**：核心写作区域。
3.  **右侧栏 (Sidebar - AI)**：AI 助手与工具箱。

### 4.2 左侧栏功能 (Left Sidebar)
*   **返回按钮**：点击返回章节管理页。
*   **章节导航树**：
    *   点击章节标题，中间编辑器滚动至该章节位置。
    *   显示章节完成状态图标。
*   **角色卡片列表**：
    *   **AI 自动生成按钮**：点击后，AI 分析大纲自动提取角色，生成姓名、人设和头像。
    *   **添加角色按钮**：手动创建新角色。
    *   **角色卡片**：点击可编辑角色详情（姓名、外貌、性格）。

### 4.3 中间编辑器功能 (Main Editor)
*   **Tiptap 编辑器**：
    *   支持 Markdown 快捷语法（# 标题，* 列表）。
    *   **智能续写**：光标停止输入 2 秒后，若开启“自动续写”，显示灰色建议文字，按 Tab 键上屏。
*   **顶部工具栏**：
    *   **章节标题**：显示当前编辑的章节。
    *   **字数统计**：实时更新。
    *   **AI 状态指示器**：显示“AI 思考中”、“撰写中”或“空闲”。
*   **悬浮气泡菜单 (Bubble Menu)**：
    *   *触发*：选中一段文本时自动弹出。
    *   *功能按钮*：
        *   **润色**：优化选中文本的文笔。
        *   **扩写**：将简短描述扩展为详细描写。
        *   **改写**：换一种表达方式。
        *   **续写**：基于选中文本继续创作。

### 4.4 右侧栏功能 (Right Sidebar)
*   **Tab 切换**：支持 `Chat` (对话) 和 `Script` (剧本工具) 两个面板。
*   **Chat 面板**：
    *   聊天窗口：与 AI 进行自由对话，询问剧情建议。
    *   上下文感知：AI 自动读取当前章节内容作为 Context。
*   **Script 面板**：
    *   **五感填充**：点击后弹出对话框，输入平淡的句子，AI 生成视/听/嗅/味/触五种描写。
    *   **冲突升级**：分析当前场景，提供 3 个增加冲突的建议。

## 5. 交互细节与反馈 (Micro-interactions)

1.  **Loading 状态**：
    *   所有 AI 生成按钮点击后，必须立即变为 `Loading` 状态（转圈或变灰）。
    *   生成过程中，若时间较长（>3秒），显示进度条或趣味文案（如“AI 正在构思惊天反转...”）。
2.  **错误处理**：
    *   API 请求失败时，弹出 `ElMessage.error` 提示，并允许用户重试。
3.  **自动保存**：
    *   编辑器内容变化时，防抖（Debounce 1000ms）触发自动保存，并在顶部显示“已保存”图标。
